CalltoConvey API Reference

The CalltoConvey REST API lets you integrate campaign automation, data management, and delivery analytics into your own systems and workflows.

Authentication

All API requests must be made over HTTPS. Authentication is handled via Flask-Login session cookies for browser-based integrations.

Note: The CalltoConvey API is currently session-authenticated. All endpoints are protected by @login_required and role-based access decorators. A public API key system is on the Enterprise roadmap.

Base URL: http://your-domain.com/api

All responses return JSON with:
{
  "status": "success" | "error",
  "data": { ... },
  "message": "..."
}

GET /api/notifications

GET /api/notifications
Returns a list of in-app notifications for the currently authenticated user. Includes campaign alerts, system messages, and admin broadcasts.
Response 200:
{
  "notifications": [
    {
      "id": 42,
      "message": "Campaign 'Fee Reminder' completed",
      "type": "campaign_complete",
      "is_read": false,
      "created_at": "2026-05-22T04:50:22Z"
    }
  ],
  "unread_count": 3
}

POST /api/campaigns/launch

POST /api/campaigns/launch
Launches a new outbound communication campaign for the authenticated organisation. Requires campaign configuration and a valid module with records.
ParameterTypeRequiredDescription
module_idintegerrequiredID of the data module to use
channelstringrequiredvoice or whatsapp
script_idintegerrequiredID of the message script template
worker_idintegeroptionalAssign campaign to a specific worker
namestringoptionalCampaign display name

POST /api/modules/<id>/import

POST /api/modules/<module_id>/import
Imports records into a module from a multipart form upload. Accepts .xlsx and .csv files. Column names must match module field names.
ParameterTypeRequiredDescription
filefilerequiredExcel (.xlsx) or CSV file with records

POST /webhooks/twilio/status

POST /webhooks/twilio/status
Twilio calls this endpoint automatically to update delivery status. Set this as your Twilio Status Callback URL in your campaign configuration. This endpoint is public (no auth required).
Twilio POSTs form data:
CallSid = "CA..."
CallStatus = "completed" | "busy" | "no-answer" | "failed"
To = "+91XXXXXXXXXX"
CallDuration = 38
